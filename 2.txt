### Key Insights

1. **Railway Cache is Stubborn**: Even after pushing code changes, Railway may use cached Docker layers. Use environment variables or Dockerfile changes to force rebuild.

2. **Dependency Conflicts**: Python packages often have conflicting version requirements. Use flexible versions (`>=`) when possible.

3. **Dockerfile Context**: Railway may use different build contexts. Keep COPY paths simple (use `.` for current directory).

4. **Web App Needs Own Dependencies**: Next.js apps in monorepos need their own `node_modules` - can't rely on root dependencies.
