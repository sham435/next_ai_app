## Current Session (March 16, 2026) - Session 2

### Issues Encountered

#### 1. Scraper Service - Playwright Version Conflict
- **Problem**: `crawl4ai==0.4.1` requires `playwright>=1.49.0` but requirements had `playwright==1.48.0`
- **Error**: `ERROR: Cannot install -r requirements.txt (line 10) and playwright==1.48.0 because these package versions have conflicting dependencies`
- **Fix Attempt 1**: Changed `playwright==1.48.0` to `playwright>=1.49.0`
- **Issue**: Railway kept using cached old version (playwright==1.48.0) even after pushing changes

#### 2. Railway Cache Issue
- Railway was using cached Docker layers from old builds
- Even after pushing new commits, the build still failed with old requirements.txt content
- **Attempted fixes**:
  - Added BUILD_DATE environment variable to force rebuild
  - Modified Dockerfile comments to invalidate cache
  - Used `railway up --detach` to trigger fresh builds
  - Multiple redeploys

#### 3. Python Syntax Error in main.py
- **Problem**: F-string syntax error at line 427 in streaming response
- **Error**: `SyntaxError: unterminated string literal`
- **Code**:
  ```python
  yield f"data: {json.dumps({
      'url': result.url,
      ...
  })}\n\n"
  ```
- **Fix**: Refactored to use intermediate variable:
  ```python
  data = {
      'url': result.url,
      ...
  }
  yield f"data: {json.dumps(data)}\n\n"
  ```

#### 4. pydantic Version Conflict
- **Problem**: `fastapi==0.104.1` requires `pydantic!=1.8, !=1.8.1, !=2.0.0, !=2.1.0, <3.0.0 and >=1.7.4` but `crawl4ai` requires `pydantic>=2.10`
- **Fix**: Changed `pydantic==2.4.2` to `pydantic>=2.10`

#### 5. Dockerfile COPY Path Issues
- **Problem**: Railway couldn't find `/apps/scraper-service` in build context
- **Error**: `failed to compute cache key: failed to calculate checksum: "/apps/scraper-service": not found`
- **Root Cause**: Railway was using different build context than expected
- **Fix**: Changed Dockerfile COPY paths from `apps/scraper-service/` to `.`

#### 6. Web Service - next not found
- **Problem**: `sh: next: not found` during npm run build
- **Root Cause**: Node modules weren't being installed properly - the web app needs its own node_modules
- **Fix**: Updated Dockerfile to install web app dependencies separately
